# 📘 SQL Production Handbook (Part I + Part II)

A practical, real-world focused SQL handbook designed to help you move from **basic querying → production-level SQL thinking**.

This repository combines:
- Fundamentals (Part I)
- Real-world production practices (Part II)

---

## 🎯 Built For

- Data Analysts  
- Business Analysts  
- Data Engineers  
- Anyone working with SQL in real projects  

---

## 🚀 What You’ll Learn

This handbook is structured in **layers + phases**, just like how SQL is actually used in real jobs.

---

# 🧱 PART I — Core SQL Foundations

## 🔹 Layer 1: Query & Analysis (Daily Use)

**Focus:** Writing efficient queries for day-to-day work

- SQL Classification  
- Joins (Inner, Left, Right, Full)  
- Clauses (WHERE, GROUP BY, HAVING, ORDER BY)  
- Window Functions  
- CTE (Common Table Expressions)  
- Date & Time Functions  
- Views  
- Primary Key vs Foreign Key  

---

## 🔹 Layer 2: Database Concepts (Design + Thinking)

**Focus:** Understanding how databases actually work

- Normalization  
- Denormalization  
- Indexes  
- Transactions  
- ACID Properties  
- Constraints (NOT NULL, UNIQUE, CHECK)  

---

## 🔹 Layer 3: Procedural & Advanced SQL

**Focus:** Advanced features used in production systems

- Stored Procedures  
- Triggers  
- Cursors  
- User Defined Functions (UDFs)  
- Temporary Tables  
- Dynamic SQL  

---

# ⚙️ PART II — Production SQL Practices

## 🔹 Phase 1: Data Exploration & Safety

**Focus:** Working safely with real production data

- Safe Data Deletion & Archiving  
- Data Profiling & Defensive Querying  
- Data Cleaning & Handling Anomalies  
- Data Validation & Quality Checks  

---

## 🔹 Phase 2: Scalable & Maintainable SQL

**Focus:** Writing SQL that scales and is easy to maintain

- Modular SQL (CTEs, Temp Tables, Views)  
- Advanced Window Functions  
- Complex Joins & Star Schema Navigation  

---

## 💡 Why This Handbook?

Most SQL resources teach syntax.

This handbook focuses on:
- How SQL is used in **real production environments**
- Avoiding common mistakes (like **fan-out, bad joins, unsafe deletes**)
- Writing SQL that is:
  - Scalable  
  - Readable  
  - Maintainable
